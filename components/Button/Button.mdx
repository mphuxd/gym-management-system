## Component API

<ArgsTable of={Button} />

### Button `as`

`as` allows you to specify a different HTML element to be rendered as a
`Button`.

### Button `className`

`className` allows the user to pass custom class names for greater styling flexibility.

### Button `href`

`href` specifies an address to navigate to on click. This changes the `Button` component to be rendered as an `a` anchor element using next/link `Link` component.

### Button `intent`

`intent` specifies which button variant to render. Possible options are `primary`, `secondary`, `tertiary`, `neutral`, `danger`, `dark`, `disabled`, and `ghost`.

### Button `role`

If you use the [as](#button-as) prop to render a non-button html element as a button, you may need to add a `role` for accessibility reasons. Adding `role="button"` will make
an element appear as a button control to a screen reader.

### Button `size`

This prop specifies which size the `Button` should be rendered at by modifying vertical and horizontal padding. If no `size` is specified, `size` is defaulted to `base`.

### Button `length`

This prop specifies which length the `Button` should be rendered at by changing button width. If no `length` is specified, `length` is defaulted to `auto`.
